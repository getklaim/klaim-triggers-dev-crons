## Problem

No `.claudeignore` file exists at the repo root. When Claude runs a broad glob pattern
such as `**/klaim-trigger-dev-crons/**`, the result set is dominated by `node_modules`
entries and gets truncated before reaching actual source files. This was observed in
session 54fecdfa, where the glob was used to survey the codebase and hit node_modules,
causing result truncation.

## Evidence

- `.claudeignore` absent: confirmed via Glob search returning no results.
- Glob `*` at repo root returned 100+ `node_modules/**` entries before any `src/` file
  appeared, and results were cut off by the tool's output limit.
- The entire source tree is only 8 `.ts` files under `src/`, meaning node_modules
  represents close to 100% of glob noise.

## Suggested fix

Create `/Users/dgsw68/Desktop/klaim/klaim-triggers-dev-crons/.claudeignore`:

```
node_modules/
dist/
.git/
```

This ensures any glob or file-search Claude runs stays within the 8-file source tree
and avoids truncation that masks relevant files.

## Impact

Every broad codebase survey in this repo currently wastes tool calls iterating
node_modules. A single `.claudeignore` file eliminates this permanently.
